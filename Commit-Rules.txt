Use this format for every commit:
<type>(<scope>): <short description>

Types that will be used in this project:
feat      - A new feature or pipeline addition
fix       - A bug fix
data      - Data processing scripts, augmentation, cleaning
train     - Training loop, config, scheduler changes
model     - Model architecture or checkpoint changes
refactor  - Code restructuring without behavior change
docs      - README, comments, documentation
chore     - Dependencies, .gitignore, CI setup
test      - Unit tests or evaluation scripts

Scopes:
data-proc     - Data processing pipeline
synth-data    - Synthetic printed data generation pipeline
interface     - Interface pipeline 
training      - Core training logic
config        - Config files

Examples:
feat(synth-data): add font renderer for Vietnamese diacritics
fix(data-proc): handle corrupted image files in loader
train(training): add cosine LR scheduler with warmup
data(data-proc): implement train/val/test split script
chore: add .gitignore for datasets and model checkpoints
docs: update README with training instructions
refactor(synth-data): separate background generator into module

Rules:
Always lowercase
No period at the end
Keep description under 72 characters
Use imperative mood: "add", "fix", "update" — not "added" or "adding"
